# 实验三：幻觉对照

## 目标

1. 对比无材料、提供材料、提供材料并要求无依据拒答三种 Prompt 的回答差异。
2. 记录模型是否编造、是否引用材料、是否在证据不足时拒答。
3. 理解 Prompt 和材料只能降低幻觉风险，不能提供绝对事实保证。

## 准备

- 执行：`export DEEPSEEK_API_KEY='你的密钥'`。
- 安装 `curl` 和 `jq`。
- 本实验使用完全虚构的“星河购”退款规则，不能把模型回答当成任何真实公司的政策。
- 三组请求使用相同模型、相同问题和相同 `max_tokens`，每组执行 3 次。
- 人工判定标准：回答中出现材料未给出的天数、金额、入口、责任或承诺，标记为“编造”。

## 逐步操作

1. 执行 A 组“无材料”请求，观察模型是否自行补全公司政策。
2. 执行 B 组“提供材料”请求，观察答案是否更贴近材料、是否指出材料缺口。
3. 执行 C 组“材料 + 无依据拒答 + 引用编号”请求，观察是否能拒绝回答材料外问题。
4. 每组执行 3 次，不能只挑表现最好的一次。
5. 按记录表逐项检查：是否编造、是否引用、是否拒答、答案是否与材料一致。
6. 总结剩余风险，例如伪引用、错误归纳、把申请条件说成退款成功承诺。

## 可复制请求或调用方式

定义通用调用函数：

```bash
ask_deepseek() {
  local prompt="$1"
  jq -n --arg prompt "$prompt" \
    '{model:"deepseek-v4-flash",thinking:{type:"disabled"},messages:[{role:"user",content:$prompt}],max_tokens:500}' |
    curl -sS https://api.deepseek.com/chat/completions \
      -H "Authorization: Bearer $DEEPSEEK_API_KEY" \
      -H 'Content-Type: application/json' \
      -d @- |
    jq '{content:.choices[0].message.content,usage:.usage,finish_reason:.choices[0].finish_reason}'
}
```

复制 A 组请求：

```bash
PROMPT_A='请回答用户关于星河购退款的问题。回答要明确、简洁。

用户问题：我在星河购买的定制水杯已经签收10天，没有质量问题。请告诉我能否退款、多久到账，并给出客服电话。'
for RUN in 1 2 3; do echo "A-$RUN"; ask_deepseek "$PROMPT_A"; done
```

## 预期观察

- A 组缺少事实材料，最容易根据常识编造退款时间、客服电话或例外政策。
- B 组通常更贴近材料，但仅写“参考材料”仍可能让模型补全材料外信息。
- C 组通过限定证据、要求引用和无依据拒答，通常能进一步降低编造风险。
- 引用编号不代表结论必然正确，仍需检查引用是否真的支持该结论。
- 即使 C 组表现正确，也不能证明以后每次都正确；Prompt 约束不是权限系统、规则引擎或事实数据库。

## 结论

降低幻觉需要“提供可信材料 + 限定只能基于材料 + 无依据拒答 + 引用 + 后端复核”的组合。Prompt 可以改善行为，但不能把概率生成模型变成绝对可靠的事实接口。退款等真实业务结果还必须查询订单、规则和审核状态，不能由模型自行承诺。

## 自测

1. 为什么提供材料后仍可能出现幻觉？
2. “带引用”为什么不等于“引用正确”？
3. 本实验中哪些内容必须回答“材料未提供，无法确认”？
4. 为什么模型不能直接承诺退款成功？
5. 生产系统还需要哪些工程措施降低幻觉风险？

## 面试表达

> 我用一份明确标注为虚构的退款规则做了三组对照。无材料时，模型容易自行补全到账时间和客服电话；加入材料后答案更贴近证据；再增加“只能基于材料、无依据拒答、逐条引用”后，编造风险进一步下降。但我不会把 Prompt 当成事实保证，后端还要校验引用、查询真实订单和规则，高风险结论需要规则系统或人工确认。
