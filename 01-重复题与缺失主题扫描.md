# 重复题与缺失主题扫描

## 1. 扫描结论

本次扫描范围为 interview 知识库下的 Markdown 面试资料，排除了已生成的 `00-总目录与知识点索引.md` 目录页，重点检查 Java 基础、并发、Spring、MySQL、Redis、RocketMQ、业务场景、项目经验和简历材料之间的主题重叠与主题缺口。

整体覆盖已经比较完整：Java 基础、并发、Spring、MySQL、Redis、RocketMQ、互联网业务场景、项目经验表达都有成体系内容。当前主要问题不是内容太少，而是部分高频主题在“原理篇、场景篇、项目篇”之间重复出现，需要明确主文档和引用关系。

建议按三类维护：

| 类型 | 定位 | 处理方式 |
|---|---|---|
| 原理篇 | 讲底层机制、关键概念、核心流程 | 保留最完整解释，作为主文档 |
| 场景篇 | 讲业务落地、方案取舍、线上排查 | 保留面试答法，引用原理篇 |
| 项目表达篇 | 讲自己项目中的实际应用 | 保留项目叙述，引用原理和场景 |

## 5. 建议整理策略

### 5.1 不建议直接删除重复题

这些重复题大多不是无效重复，而是同一个主题在不同面试语境下的表达：

- 原理面试：问“为什么、底层怎么做”。
- 场景面试：问“业务中怎么设计”。
- 项目面试：问“你实际做过什么”。

直接删除会损失面试表达层次，更好的方式是保留主文档，其他文档改为摘要和链接。

### 5.2 建议建立主文档映射

| 主题 | 主文档 | 其他文档处理方式 |
|---|---|---|
| Redis | `Java高级开发查缺补漏.md` | 业务文档保留场景答法 |
| JVM | `Java JVM高频面试题与线上排障指南.md` | 核心基础保留类加载/String 上下文，并发保留 JMM/ThreadLocal/线程池上下文，业务文档保留 OOM 场景答法，高级补漏只保留速记入口 |
| RocketMQ | `RocketMQ高频面试题.md` | 高级补漏保留速记，项目文档保留经历 |
| Spring | `Spring核心原理面试指南.md` | 业务文档保留口语化速答 |
