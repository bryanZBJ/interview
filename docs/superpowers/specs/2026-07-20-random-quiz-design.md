# 面试知识库随机出题功能设计

## 1. 目标

在现有单 HTML 面试学习站中增加一个不依赖后端和大模型的随机练习功能，帮助用户从全部已发布知识点中随机抽题，先回忆、再查看原文答案，并继续使用现有的学习状态。

成功标准：

1. 桌面侧边栏和手机底部导航都有“练习”入口。
2. 题目默认覆盖全部知识库，不受知识库当前专题筛选影响。
3. 题目先显示，答案只在用户点击“查看答案”后展开。
4. 一轮内不重复出题，题库用完后自动重新洗牌。
5. 可以在练习页将当前知识点标记为“未学习 / 需复习 / 已掌握”，与阅读页、复习清单同步。
6. 保持单 HTML、纯静态、手机与 macOS 兼容的现有架构。

## 2. 范围

### 2.1. 本次包含

- 新增 `#/quiz` 内部路由。
- 主导航新增“练习”入口。
- 基于标题的题目适用性筛选。
- 一轮内无重复的随机洗牌队列。
- 题目、专题和文档来源展示。
- 点击后展开当前知识点对应的完整章节答案。
- 现有三态学习标记和 `localStorage` 复用。
- 桌面与移动端响应式布局、键盘操作和可访问性验证。

### 2.2. 本次不包含

- 多选题、干扰项和自动判分。
- 调用 LLM 生成新题目或评价用户回答。
- 用户输入答案、语音回答或录音。
- 云端同步题目进度、账号、积分、排行榜。
- 间隔重复算法、每日任务或练习历史报表。
- 按专题、状态或难度自定义抽题范围。

## 3. 用户流程

```text
点击“练习”
  -> 构建全库合格题目集
  -> 洗牌并取出第一题
  -> 显示题目，隐藏答案
  -> 用户自行回忆
  -> 点击“查看答案”
  -> 展开对应文档章节
  -> 可选标记学习状态
  -> 点击“下一题”
  -> 从当前洗牌队列取下一题
```

刷新页面后重新构建和洗牌题库，不恢复上一次题序。学习状态按现有规则持久化。

## 4. 题库与智能筛选

### 4.1. 数据源

题库仅使用已进入 `site.config.mjs` 发布允许列表的 `data.points`。不扫描额外文件，不扩大公开站点的内容范围。

每道题使用现有知识点数据：

- `id`：用于学习状态和题序。
- `title`：用于生成题目。
- `headingId`：用于从文档 HTML 提取答案章节。
- `documentSlug`：用于找到来源文档。
- `topic`：用于显示题目专题。
- `excerpt`：章节提取失败时的降级答案。

### 4.2. 合格题目

知识点必须具有非空 `title`、`excerpt`、`documentSlug` 和 `headingId`。

标题符合以下任一特征时优先作为题目：

- 以 `Q`、`追问`、`问题` 等形式开头。
- 包含问号。
- 是具体技术概念、机制、流程、比较或业务场景标题。

下列结构性标题不进入题库：

- 使用说明、阅读说明、学习目标、必须理解的概念。
- 核心结论、生产建议、面试口述、今日产出、完成打卡。
- 参考、参考资料、延伸阅读、实验原文。
- 跟着做、阅读项目代码、修改实验、记录表、预期观察。
- 快速验收、不看答案自测、自测、总结等组织性标题。

筛选采用小型、明确的黑名单匹配，不引入 LLM 分类，不依赖网络。

### 4.3. 题目文案

- 标题已包含问号，或以 `Q`、`追问`、`问题` 开头：保留原标题。
- 其他概念型标题：显示为 `请解释：{title}`。
- 题目上方显示专题和来源文档，但不预先展示正文摘要。

## 5. 随机与去重

进入练习页时，使用 Fisher-Yates 算法对合格题目 ID 进行洗牌。

页面内存中保存：

```text
quizQueue       当前轮剩余题目 ID
currentQuizId   当前题目 ID
answerRevealed  当前答案是否已展开
```

点击“下一题”时从队列取出一题。队列为空时重新洗牌，并在题库多于一题时避免新一轮第一题与上一轮最后一题相同。

不将题序写入 `localStorage`，避免增加状态迁移和旧内容变更后的无效 ID 处理。

## 6. 答案提取

不在构建数据中复制每个知识点的完整答案，避免让单 HTML 的体积明显增长。

浏览器展开答案时：

1. 从 `documentMap` 找到当前文档 HTML。
2. 使用 `DOMParser` 解析为脱离当前页面的 DOM。
3. 通过 `headingId` 找到对应二级或三级标题。
4. 从标题后开始复制相邻节点，直到遇到同级或更高级标题。
5. 将提取内容放入答案区域。

如果找不到标题或没有可用正文，则显示经过 HTML 转义的 `excerpt`，并提供“查看原文”入口。

## 7. 页面与交互

### 7.1. 导航

- 桌面侧边栏：`首页 / 知识库 / 练习 / 复习`。
- 手机底部导航：四等分显示上述四项，不横向滚动。
- 练习页顶栏标题为“随机练习”。
- 练习导航按钮使用 Lucide `shuffle` 图标。

### 7.2. 练习页

页面从上到下包含：

1. 简短页面标题与“全部题库”范围说明。
2. 题目进度：当前轮已练习数和题库总数。
3. 题目区：专题、来源文档、题目文案。
4. 主操作：答案未显示时为“查看答案”。
5. 答案区：展示原文章节，支持代码块、列表、表格和链接。
6. 状态控件：答案展开后显示现有三段式学习状态。
7. 底部操作：“查看原文”和“下一题”。

展开答案后将焦点移到答案标题，并使用 `aria-live="polite"` 通知辅助技术。点击“下一题”后将页面回到题目顶部，不要让用户留在上一题答案底部。

### 7.3. 视觉约束

- 继续使用现有灰白底、专业蓝、绿色已掌握和琥珀色需复习语义。
- 不创建卡片套卡片；题目与答案是一个连续的练习工作区。
- 内容区标题使用紧凑尺寸，不使用营销页式大字。
- 主按钮和手机触摸目标不小于 44px。
- 题目、来源和长单词支持换行，移动端不产生水平溢出。
- 答案中的表格和代码块仅在自身容器内横向滚动。
- 支持浅色、深色、跟随系统与 `prefers-reduced-motion`。

## 8. 状态与数据流

### 8.1. 代码边界

- `src/quiz.js`：保存题目筛选、题目文案、Fisher-Yates 洗牌和章节提取等纯逻辑，并以浏览器全局对象暴露给页面脚本；该文件不读写 DOM 页面状态和 `localStorage`。
- `src/site.js`：负责 `#/quiz` 路由、队列状态、事件处理、焦点管理和学习状态联动。
- `src/template.mjs`：增加练习导航入口，并在 `site.js` 前内联 `quiz.js`，最终产物仍只有 `site/index.html`。
- `src/site.css`：增加练习页及四项移动导航样式。
- `tests/quiz.test.mjs`：直接验证纯逻辑；`tests/site.spec.mjs` 验证完整浏览器交互。

这样可以让随机与答案边界逻辑独立测试，同时不增加线上脚本请求或额外部署文件。

### 8.2. 持久化状态

当前知识点状态仍使用：

```text
interview-learning-progress-v1
```

练习页复用 `pointStatus` 和 `setPointStatus`，不增加新的持久化数据结构。设置状态后不更换当前题目，只更新控件和学习数据；用户明确点击“下一题”才进入下一题。

数据流：

```text
site-data.documents + site-data.points
              -> 题目筛选
              -> 随机洗牌队列
              -> 当前题目
              -> DOMParser 提取原文章节
              -> 答案展示

当前题目 ID
              -> pointStatus / setPointStatus
              -> localStorage
              -> 阅读页与复习页共享
```

## 9. 异常和边界

- 合格题库为空：显示空状态和“返回知识库”。
- 题库只有一题：允许下一轮再次出现，不陷入避重循环。
- 来源文档不存在：跳过该题并继续取题；如果没有其他题则显示空状态。
- 章节提取失败：显示 `excerpt` 作为降级内容，同时保留“查看原文”。
- `localStorage` 不可用：继续允许随机练习，并沿用现有状态保存失败提示。
- 在答案未展开时点击“下一题”不是必要流程，页面只在展开后显示该操作，避免误触跳过。

## 10. 测试与验收

### 10.1. 构建与单元验证

- 发布允许列表和知识点 ID 仍唯一。
- 题目筛选排除已列出的结构性标题，保留显式问句和技术概念标题。
- 洗牌不丢失、不重复题目，队列用完后能重建。
- 章节提取遇到同级或更高级标题时停止，不泄露后续题目答案。

### 10.2. Playwright 交互验证

- 桌面和手机导航都显示“练习”，并正确进入 `#/quiz`。
- 初始只显示题目，不显示章节答案。
- 点击“查看答案”后显示完整章节、学习状态和后续操作。
- 标记“需复习”后刷新，对应知识点状态仍保留。
- 点击“下一题”会更换题目并重新隐藏答案。
- 桌面和手机视口不产生水平溢出，长题目、表格和代码块可用。
- 键盘可到达所有操作，焦点转移可预期，深色模式和减少动画设置不受影响。

### 10.3. 发布验证

- `npm test`、`npm run build` 和 `npm run test:e2e` 通过。
- `site/index.html` 仍为唯一发布产物，不依赖外部运行时资源。
- GitHub Pages 发布后，手机和 macOS 可访问练习页并保留本机学习状态。

## 11. 安全与提交边界

- 本功能不读取、不存储、不发布任何 API Key。
- 提交时只包含设计、实现、测试和构建产物相关文件。
- `java-demo/src/main/resources/application.yml` 中的本地敏感配置不得进入任何提交。
- 提交前检查暂存区文件列表，并对暂存内容执行 API Key 模式扫描。
