# 单 HTML 面试学习笔记网站设计

## 1. 目标

将 `interview` 仓库中经过允许列表筛选的中文面试资料构建成一个可直接部署的 `index.html`，用于手机和 macOS 日常阅读、搜索和复习。

第一版成功标准：

1. 最终产物只有一个 `site/index.html`，不需要数据库、后端服务或账号系统。
2. 支持专题目录、中文搜索、文档阅读和知识点学习状态。
3. 学习状态保存在当前浏览器，刷新页面和更新网站内容后仍保留。
4. Markdown 推送到私有 GitHub 仓库后，GitHub Actions 自动重新生成并发布 HTML。
5. 支持 macOS Safari/Chrome、iPhone Safari 和 Android Chrome。

## 2. 范围

### 2.1. 第一版包含

- 一个自包含 HTML 文件，正文、样式、脚本、图标和搜索数据全部内嵌。
- 首页学习概览。
- 专题知识库和文档阅读视图。
- 中文标题、正文和代码片段搜索。
- `未学习 / 需复习 / 已掌握` 三种状态。
- 当前设备的总体掌握度、专题进度和“需复习”列表。
- 最近阅读知识点和继续学习入口。
- 浅色、深色和跟随系统主题。
- GitHub Pages 静态托管和推送后自动发布。

### 2.2. 第一版不包含

- 手机与 Mac 自动同步学习状态。
- D1、Worker、MySQL、Supabase 等云端存储。
- 用户注册、登录、私密链接或访问密码。
- 网页编辑 Markdown。
- 多用户、评论、分享和公开进度。
- AI 问答、自动出题和间隔重复算法。
- 完整离线 PWA 安装。

## 3. 访问与隐私边界

网站部署为公开静态页面，任何获得网址的人都可以访问。用户不会主动分享网址，但该方案不提供真正的权限保护。

私有 GitHub 仓库只保护源码仓库，不等于 GitHub Pages 网站私有。发布允许列表是防止简历和个人资料进入 HTML 的核心安全边界。

## 4. 内容范围

构建脚本使用显式允许列表，不扫描并发布全部文件。

默认包含：

- Java 核心基础、JVM、并发、IO。
- Spring、MySQL、Elasticsearch、Kafka、RocketMQ。
- 分布式组件、微服务治理、Docker / Kubernetes、可观测性、安全认证。
- Java 互联网业务场景、高级开发查缺补漏、面试题复盘。
- AI 应用开发、AI Agent 岗位准备。

默认排除：

- `jianli/` 目录和所有简历、PDF、Word 文件。
- `AGENTS.md`、仓库规则、总目录、重复题扫描报告。
- 个人项目经历、公司信息和其他未列入允许列表的 Markdown。
- 临时文件、预览草图和构建产物。

新增专题时必须显式加入允许列表，构建脚本才会发布。

## 5. 架构

```text
允许列表中的 Markdown
          |
          | Node.js 构建脚本
          v
      site/index.html
          |
          | GitHub Actions
          v
       GitHub Pages
          |
          +---- 手机浏览器 localStorage
          +---- Mac 浏览器 localStorage
```

手机和 Mac 都访问相同 HTML，但各自保存独立学习状态。

## 6. 单文件生成方式

构建脚本执行以下步骤：

1. 读取允许列表中的 Markdown。
2. 校验文件存在且不存在重复 slug。
3. 转换 Obsidian 文档链接和章节链接。
4. 将 Markdown 转换成经过清理的 HTML。
5. 从二级、三级标题和 `Q`/`追问` 标题中提取知识点。
6. 生成文档目录、知识点列表和纯文本搜索数据。
7. 将内容数据安全编码进 `<script type="application/json">`。
8. 将 CSS、JavaScript 和 Lucide 图标内嵌到 HTML。
9. 输出 `site/index.html` 和构建摘要。

最终 HTML 不依赖 CDN、外部字体、外部 JavaScript 或运行时接口。只要浏览器能打开页面，正文和功能就能加载。

## 7. 知识点与本地状态

### 7.1. 知识点识别

默认识别为知识点：

- `Q1`、`Q2` 等题目标题。
- 包含“追问”的标题。
- 二级、三级标题下具有正文的章节。

纯分组标题不计入掌握度。

### 7.2. 知识点 ID

知识点 ID 由以下内容生成：

```text
文档 slug + 标题层级路径
```

因此：

- 只修改正文不会影响原有状态。
- 在标题下增加内容不会影响原有状态。
- 重命名标题或移动到不同父标题下，会被视为新知识点。
- 删除知识点不会立即清理本地旧状态，避免误删后无法恢复。

第一版不维护 UUID 注册表，以换取更简单的构建和维护方式。

### 7.3. localStorage 数据

使用一个版本化键：

```text
interview-learning-progress-v1
```

数据结构：

```json
{
  "jvm::1-jvm-运行时内存区域::java-内存模型": {
    "status": "review",
    "updatedAt": 1784016000000
  }
}
```

状态只能是：

- `unlearned`：未学习。
- `review`：需复习。
- `mastered`：已掌握。

未出现在 localStorage 中的知识点默认视为“未学习”。如果 localStorage 内容损坏，页面保留一份损坏数据备份并恢复为空状态，同时提示用户。

## 8. 页面结构

虽然最终只有一个 HTML 文件，但运行时包含三个内部视图和一个阅读视图，不发生页面跳转或重新加载。

### 8.1. 首页

- 需复习数量。
- 总体掌握度。
- 各专题掌握度。
- 继续上次阅读。
- 最近更新文档。

### 8.2. 知识库

- 专题与文档列表。
- 文档知识点数量和当前设备掌握度。
- 搜索框、专题筛选和状态筛选。

### 8.3. 阅读视图

- Markdown 正文。
- 页内目录。
- 当前知识点三状态控件。
- 上一知识点、下一知识点。
- 自动记录最后阅读文档和标题锚点。

### 8.4. 复习视图

- 只展示当前设备标记为“需复习”的知识点。
- 支持专题筛选。
- 支持按文档顺序开始复习。
- 没有待复习内容时展示明确空状态。

## 9. 搜索

第一版使用内置 JavaScript 搜索，不引入运行时搜索服务。

搜索规则：

- 标题完全命中权重最高。
- 标题包含匹配高于正文匹配。
- 支持中文连续字符、英文大小写忽略和代码关键词。
- 搜索结果显示专题、文档、标题和正文摘要。
- 输入 150ms 后执行搜索，避免每次按键都扫描全部内容。
- 无结果时提供清除筛选和推荐关键词。

如果知识点数量增长导致搜索明显卡顿，再考虑将轻量搜索库打包进 HTML，第一版不提前增加依赖。

## 10. 视觉与交互

沿用已确认的 `Knowledge Studio`：

- 灰白背景、深灰正文、专业蓝主操作。
- 绿色表示已掌握，琥珀色表示需复习，同时始终显示状态文字。
- 卡片圆角不超过 8px，不使用卡片套卡片。
- 手机正文至少 16px，行高不低于 1.6。
- 桌面正文最大宽度约 72 个字符。
- 交互目标不小于 44px。
- 使用系统中文字体，不依赖 Google Fonts。
- Lucide 图标在构建阶段内嵌，不手绘 SVG。
- 动画只用于视图切换和状态反馈，并遵循 `prefers-reduced-motion`。

导航方式：

- 手机：底部 `首页 / 知识库 / 复习`。
- macOS：左侧专题导航和顶部搜索。
- `Command + K` 打开搜索，Windows/Linux 使用 `Ctrl + K`。
- 浏览器前进、后退通过 URL hash 恢复视图和阅读锚点。

## 11. macOS 与移动端兼容

正式验收：

- macOS Safari 当前稳定版。
- macOS Chrome 当前稳定版。
- iPhone Safari 当前稳定版。
- Android Chrome 当前稳定版。
- 375px、768px、1280px、1440px 视口。

兼容要求：

- 支持鼠标、触控板、键盘和触摸操作。
- 关键功能不依赖悬停。
- 支持浏览器缩放，不禁止页面缩放。
- 支持系统深色模式和减少动画。
- 表格和代码块在自身容器内横向滚动，不撑宽页面。
- Safari 和 Chrome 都能读写 localStorage，并在网站更新后保留状态。

## 12. GitHub 自动发布

GitHub Actions 在以下文件变化时运行：

- 允许列表中的 Markdown。
- HTML 模板、样式或脚本源码。
- 构建脚本和允许列表配置。

流水线：

```text
安装依赖 -> 内容校验 -> 自动测试 -> 生成 index.html -> 检查单文件 -> 发布 GitHub Pages
```

构建产物要求：

- 只能产生一个对外页面 `site/index.html`。
- 不包含简历和排除文档的标题或正文。
- 不包含本地绝对路径、环境变量和仓库凭据。
- HTML 小于 8 MB 时正常发布；达到 8 MB 给出警告；超过 15 MB 阻止发布并要求拆分方案。

构建或测试失败时，GitHub Pages 继续保留上一个成功版本。

## 13. 错误处理

| 场景 | 处理方式 |
|---|---|
| 允许列表文件不存在 | 构建失败，不发布 |
| 重复文档 slug 或知识点 ID | 构建失败并列出冲突 |
| Obsidian 链接目标不存在 | 构建报告列出来源；核心导航链接失效时阻止发布 |
| Markdown 解析失败 | 构建失败并输出文件名 |
| localStorage 不可用 | 页面仍可阅读和搜索，状态功能显示当前浏览器无法保存 |
| localStorage 数据损坏 | 备份损坏值、恢复空状态并提示 |
| 搜索无结果 | 展示推荐关键词和清除筛选操作 |
| HTML 超过 15 MB | 阻止发布，重新评估多文件方案 |

## 14. 测试与验收

### 14.1. 构建测试

- 允许列表只包含批准的面试专题。
- Markdown 标题、表格、代码块、引用和中文锚点正确转换。
- Obsidian 链接正确转换并检测失效目标。
- 知识点 ID 在只修改正文时保持不变。
- 排除文档内容不出现在 HTML。
- 内嵌 JSON 不会通过 `</script>` 提前结束脚本标签。
- 产物只有一个 HTML，且不包含外部运行时依赖。

### 14.2. 浏览器测试

- 首页、知识库、阅读和复习视图切换。
- 中文搜索、专题筛选、状态筛选。
- 三种状态保存、刷新恢复和网站内容更新后恢复。
- `Command + K`、Tab、Enter、Space 和 Escape。
- 手机底部导航、macOS 左侧导航。
- 浅色、深色和减少动画。
- 375px、768px、1280px、1440px 无页面横向溢出。

### 14.3. 发布验收

1. GitHub Pages URL 在手机和 macOS 可访问。
2. 手机标记的状态刷新后保留。
3. Mac 标记的状态刷新后保留。
4. 手机和 Mac 状态互不覆盖、互不同步，符合精简方案边界。
5. 修改一处 Markdown 后推送，线上 HTML 自动更新，原设备状态仍保留。
6. 页面源码中不存在简历内容、本地路径或密钥。

## 15. 日常使用

更新知识正文：

```text
修改 Markdown -> git commit -> git push -> GitHub 自动重新发布
```

新增专题：

```text
新增 Markdown -> 加入允许列表 -> 本地构建检查 -> git push
```

学习进度只存在当前浏览器。清除浏览器网站数据、更换浏览器或更换设备会丢失该设备进度，第一版接受这一边界。
