# 面试学习笔记网站设计

## 1. 背景与目标

当前 `interview` 仓库以中文 Markdown 维护 Java、JVM、并发、Spring、数据库、分布式、微服务和 AI Agent 等面试资料。目标是在不改变本地写作习惯的前提下，生成一个可在手机和 macOS 访问的私密学习网站，并支持跨设备同步学习进度。

第一版成功标准：

1. 本地 Markdown 是唯一正文来源，推送私有 GitHub 后自动更新网站。
2. 手机和 macOS 能阅读、搜索和浏览专题目录。
3. 每个知识点可标记为 `未学习`、`需复习`、`已掌握`，状态跨设备同步。
4. 不建立账号系统，通过可轮换的专属私密链接访问。
5. 文档解析或部署失败时保留上一个线上可用版本。

## 2. 范围

### 2.1. 第一版包含

- 首页学习驾驶舱。
- 专题知识库与文档阅读页。
- 中文全文搜索，覆盖标题、正文和代码片段。
- 按知识点记录三种学习状态。
- “需复习”知识点清单和顺序复习。
- 最近阅读位置与继续学习入口。
- 私有 GitHub 推送后的自动构建和发布。
- 单用户私密访问、跨设备进度同步。
- 浅色和深色主题。
- 手机、平板、macOS Safari 和 Chrome 响应式适配。

### 2.2. 第一版不包含

- 网页编辑 Markdown。
- 多用户注册、登录和权限角色。
- 评论、分享、点赞和公开个人主页。
- AI 问答、自动出题和复杂间隔记忆算法。
- 简历、个人信息、仓库规则和重复题扫描报告发布。

## 3. 内容范围

发布过程采用显式允许列表，不直接扫描并公开仓库中的所有文件。

默认包含：

- Java 核心基础、JVM、并发、IO。
- Spring、MySQL、Elasticsearch、Kafka、RocketMQ。
- 分布式组件、微服务治理、Docker / Kubernetes、可观测性、安全认证。
- Java 互联网业务场景、高级开发查缺补漏、面试题复盘。
- AI 应用开发、AI Agent 岗位准备。

默认排除：

- `jianli/` 及简历相关文件。
- `AGENTS.md`、`00-总目录与知识点索引.md`、`01-重复题与缺失主题扫描.md`。
- 具体个人项目梳理和包含个人经历、公司信息的文档。
- 临时文件、构建产物、PDF、Word 和图片渲染目录。

允许列表由站点配置文件维护，新增专题时显式加入，避免误发布敏感内容。

## 4. 技术架构

采用“静态内容构建 + Cloudflare Worker + D1”的单站点架构：

```text
本地 Markdown
      |
      | git push
      v
私有 GitHub 仓库
      |
      | GitHub Actions
      v
Markdown 解析、知识点注册、搜索索引和站点构建
      |
      v
Cloudflare Worker 站点
      |
      +---- 私密访问校验
      +---- 学习进度 API ---- D1
```

职责边界：

| 组件 | 职责 |
|---|---|
| Markdown 文件 | 唯一正文来源 |
| 构建脚本 | 解析文档、生成目录、知识点、链接和搜索索引 |
| GitHub Actions | 校验、测试、构建和自动发布 |
| Cloudflare Worker | 返回站点、校验私密访问、提供进度 API |
| D1 | 保存学习状态、最后阅读时间和阅读位置，不保存正文 |
| 浏览器本地存储 | 保存待同步状态队列和非敏感界面偏好 |

## 5. 内容解析与知识点标识

### 5.1. Markdown 解析

支持以下现有写法：

- 一级至六级标题。
- 表格、列表、引用、代码块和行内代码。
- 中文标点标题和重复标题。
- Obsidian 文档链接与章节链接。
- 普通 Markdown 链接。

构建时把文档转换为结构化内容，并生成：

- 专题与文档目录。
- 页内章节目录。
- 全文搜索索引。
- 上一知识点、下一知识点关系。
- 来源文件和标题锚点。

### 5.2. 知识点粒度

默认将以下内容识别为可标记知识点：

- 以 `Q`、`追问` 开头的题目标题。
- 二级、三级标题下具有正文内容的章节。
- 配置文件中明确指定的特殊标题。

纯分组标题不生成学习状态，避免进度统计失真。

### 5.3. 稳定知识点 ID

仓库维护一个知识点 ID 注册表。每条记录至少包含：

- 稳定 UUID。
- 来源文档路径。
- 标题层级路径。
- 规范化标题。
- 内容特征摘要。

同步规则：

1. 原路径和标题未变时直接复用 ID。
2. 正文变化但标题未变时复用 ID。
3. 标题重命名时，根据同文档位置、相邻标题和内容特征尝试匹配旧 ID。
4. 无法可靠匹配时创建新 ID，并在构建报告中列出，不静默迁移进度。
5. 删除知识点不立即删除 D1 记录，以便后续恢复或人工迁移。

## 6. 私密访问

不创建用户名和密码，采用单用户私密链接：

1. 首次访问使用包含高强度随机密钥的专属链接。
2. Worker 验证密钥后写入 `HttpOnly`、`Secure`、`SameSite=Strict` Cookie。
3. 验证成功后重定向到不包含密钥的干净地址，减少浏览器历史和复制地址时泄露密钥的风险。
4. 后续访问自动读取 Cookie，不展示登录页。
5. 密钥可以轮换；轮换后旧链接和旧会话失效。

该方案没有账号操作，但本质上仍是持有密钥即可访问。专属链接不得公开分享。

## 7. 数据模型

D1 第一版只需要单用户进度表：

```text
learning_progress
- knowledge_point_id  TEXT PRIMARY KEY
- status              TEXT NOT NULL
- last_read_anchor    TEXT
- last_read_at        INTEGER
- updated_at          INTEGER NOT NULL
- client_revision     INTEGER NOT NULL
```

约束：

- `status` 只能是 `unlearned`、`review`、`mastered`。
- 默认未写入 D1 的知识点视为 `unlearned`。
- 使用 `updated_at + client_revision` 处理多设备覆盖，后写入的有效版本生效。
- 正文、搜索索引和文档目录不进入 D1。

## 8. 页面与交互

视觉方向采用已确认的 `Knowledge Studio`：中性灰白背景、专业蓝主操作、绿色表示已掌握、琥珀色表示需复习；深色模式保持同一语义层级。

### 8.1. 首页

- 今日需复习数量。
- 总体掌握度和各专题进度。
- 继续上次阅读。
- “开始今日复习”主操作。
- 最近学习与最近更新专题。

### 8.2. 知识库

- 专题目录和每个专题的知识点数量、掌握度。
- 全文搜索与即时建议。
- 按状态、专题筛选。
- 桌面使用侧边目录，手机使用单列列表。

### 8.3. 阅读页

- Markdown 正文、来源路径和更新时间。
- 页内目录、上一篇、下一篇。
- `未学习 / 需复习 / 已掌握` 分段控件。
- 自动记录最后阅读知识点和锚点。
- 桌面正文最大宽度约 72 个字符，避免长行难读。

### 8.4. 复习页

- 默认展示所有“需复习”知识点。
- 支持按专题筛选和顺序复习。
- 阅读后可直接改为“已掌握”或继续保留“需复习”。

### 8.5. 导航

- 手机一级导航：`首页 / 知识库 / 复习`，固定底部并保留安全区间距。
- macOS 一级导航：左侧栏或顶部栏，阅读页保持专题目录可见。
- 所有知识点拥有稳定 URL，支持浏览器前进、后退和书签。
- 搜索快捷键为 `Command + K`，Windows/Linux 同时支持 `Ctrl + K`。

## 9. macOS 与多端兼容

正式验收环境：

- macOS Safari 当前稳定版。
- macOS Chrome 当前稳定版。
- iPhone Safari 和 Android Chrome 当前稳定版。
- 375px 手机、768px 平板、1280px 与 1440px 桌面宽度。

兼容要求：

- 使用系统中文字体优先，避免依赖境外字体服务才能正常显示。
- 支持鼠标、触控板、键盘和触摸操作；关键功能不依赖悬停。
- 支持浏览器缩放和系统文字放大，不禁止页面缩放。
- 支持 `prefers-color-scheme`、`prefers-reduced-motion`。
- 可见键盘焦点，交互目标不小于 44px。
- Safari 下验证 Cookie、历史回退、滚动位置恢复和代码块横向滚动。
- 桌面侧边栏不会挤压正文，超宽屏也限制正文行宽。

## 10. 同步与离线处理

状态更新采用“界面即时反馈 + 后台同步”：

1. 用户切换状态后立即更新界面。
2. 请求成功后显示已同步状态。
3. 断网或 API 失败时写入浏览器本地待同步队列。
4. 网络恢复或下次打开时按更新时间重试。
5. 队列未清空时显示明确的“待同步”提示，不把本地状态伪装成云端成功。

正文仍依赖在线站点。第一版不承诺完整离线阅读，只保证状态操作在短时断网时不丢失。

## 11. 错误处理

| 场景 | 处理方式 |
|---|---|
| Markdown 无法解析 | 构建失败，停止发布，保留线上旧版本 |
| Obsidian 链接失效 | 构建报告列出来源文件和链接；核心入口失效时阻止发布 |
| 知识点 ID 无法迁移 | 创建新 ID 并在报告中提示，不覆盖旧进度 |
| 搜索索引生成失败 | 停止发布，不上线缺少搜索的新版本 |
| D1 或进度 API 失败 | 本地排队并提示待同步，支持自动重试 |
| 私密密钥无效或过期 | 不返回正文与进度，提示使用新的专属链接 |
| 自动部署失败 | 线上继续运行最近一次成功版本 |

## 12. 测试与验收

### 12.1. 自动测试

- Markdown 标题、表格、代码块、引用和中文锚点解析。
- Obsidian 链接转换和失效链接检测。
- 知识点识别、ID 复用、标题重命名和删除场景。
- 搜索索引的中文标题、正文和代码命中。
- 三种状态的写入、读取、冲突覆盖和非法值拒绝。
- 私密链接换取 Cookie、未授权拒绝和密钥轮换。
- 断网队列恢复与重复提交幂等。

### 12.2. 浏览器验收

- iPhone Safari、Android Chrome、macOS Safari、macOS Chrome。
- 375px、768px、1280px、1440px 视口无横向溢出。
- 手机底部导航不遮挡正文，macOS 侧边栏不挤压阅读区。
- `Command + K`、Tab 焦点、Enter/Space 操作状态控件。
- 浅色、深色、高对比和减少动画设置。
- 手机修改状态后，macOS 刷新可看到同一状态；反向操作同样成立。

## 13. 发布与日常更新

首次发布：

1. 将 `interview` 初始化为 Git 仓库并推送到私有 GitHub 仓库。
2. 配置 Cloudflare Worker、D1 和私密访问密钥。
3. GitHub Actions 运行校验、测试、构建和发布。
4. 在手机和 macOS 完成首次私密链接验证。

后续更新：

```text
修改 Markdown -> git commit -> git push -> 自动校验并发布
```

如果校验失败，GitHub 显示具体文档和原因，线上内容保持不变。

## 14. 实施约束

- 不修改现有面试 Markdown 的主体写作格式来迁就网站。
- 不把 Cloudflare 密钥、私密链接或 GitHub 凭据提交到仓库。
- 不对排除文件做隐式发布。
- 不用颜色作为学习状态的唯一提示，状态始终带文字。
- 不使用悬停作为关键操作的唯一入口。
- 第一版优先可靠阅读和状态同步，不扩展未经确认的社交或 AI 功能。
